Plugin Development

*ARRgh's source plugins are esbuild-bundled TypeScript modules that implement a simple HTTP API. The plugin host loads them and proxies requests from the Rust server.

Plugin protocol

A plugin exposes these endpoints over HTTP:

EndpointDescription
GET /infoPlugin metadata (name, version, base_url, content_types)
GET /search?q=<query>Search for titles
GET /:source_id/chaptersList chapters for a title
GET /:source_id/chapters/:chapter_id/pagesGet image URLs for a chapter

Cloudflare-protected sources

Plugins that need to bypass Cloudflare can access the CloakBrowser stealth Chromium instance via the getBrowser() helper provided by the plugin host:

import { getBrowser } from '../plugin-host/src/browser'

const browser = await getBrowser()
const page = await browser.newPage()
await page.goto(url)
const html = await page.content()

Build and install

# Build your plugin
cd my-source
npm run build   # esbuild → dist/index.js

# Install into arrgh via Settings → Sources → Add custom source
# URL: http://your-plugin-host:port

Distributing via the plugin index

Community plugins can be listed in the plugin index so users can browse and install them from Settings. Open a PR to add your plugin.
