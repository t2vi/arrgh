**Released**: 2026-05-28

## What's new

### E2e test layer (Playwright)

Full end-to-end test suite against a real Docker Compose stack using a deterministic Fixture Plugin — no live sources, no network flakiness in CI.

**Infrastructure:**
- `plugins/fixture/` — standalone Express server implementing the Source Plugin Protocol with 4 deterministic modes (happy path, no-match, 502, empty pages)
- `docker-compose.test.yml` — test stack: same `arrgh` image, fixture replaces plugin-host, no CloakBrowser, isolated volume
- `e2e/` — Playwright suite with `globalSetup` (seeds admin via API → `storageState`), fixtures, and 4 spec files
- `.github/workflows/e2e.yml` — runs on push to `main`, deploys results to shared Allure cache on `gh-pages`
- Shared `allure-raw/` cache on `gh-pages` — unit + integration + e2e all feed one combined report at `/test-reports/`

**11 scenarios verified:**

| Scenario | Spec |
|---|---|
| Unauthenticated → redirected to login | auth |
| Login → logout → login again | auth |
| Wrong password → error shown | auth |
| Add title → appears in library | library |
| Sync progress overlay visible while building | library |
| Source match fails → Sync Warning badge shown | library |
| Queue chapter → download reaches done | download |
| Source 502 on chapter sync → sync warning shown | download |
| Empty pages → queue item shows error | download |
| Navigate to bad title → error state, no crash | discover |
| Discover page loads with search input | discover |

See ADR 0023 for full architecture decisions.

## Bug fixes

### 502 on chapter sync fails silently — no sync warning (fixes #69)

Previously, when a source's chapters endpoint returned 502, `sync_chapters` returned `Ok(0)` and the title silently ended up with 0 chapters and no amber warning badge.

**Fix:** `sync_chapters` now returns `Err("source temporarily unavailable (502)")` on 502. All three call sites (`add_manga`, `patch_title` spawn, `sync_title` spawn) insert a `sync_warnings` row in the error branch. `any_ok` is now set when a source is **matched** (not when chapter sync succeeds), so `sync_status` stays `ready` even when chapters are temporarily unavailable.

**Found via:** e2e "Fixture 502" scenario.

### `GET /api/version` required authentication (fixes #70)

`/api/version` was inside the auth-protected router, returning 401 without a token. Version info should be public — used for health checks, server-ready detection, and could be displayed on the login screen.

**Fix:** `version::router()` moved to the public (pre-auth) section. Unused `Extension(Claims)` extractor removed from the handler.

**Found via:** `globalSetup.ts` in e2e using `/api/version` as server-ready check.

### GHCR build cache SAS token expiry

The `arrgh` Docker image build (multi-platform Rust, `linux/amd64+arm64`) was taking >60 minutes, causing the GitHub Actions Cache SAS token to expire during layer export, failing the GHCR push.

**Fix:** `arrgh` switched from `type=gha,mode=max` to `type=registry` (layers cached in GHCR under `:buildcache` tag — no SAS token, no expiry). `plugin-host` and `cloakbrowser-server` switched to `mode=min`.

## Bugs found during e2e implementation (test infrastructure, not product)

During e2e bringup, several issues were caught and fixed before CI ran:

- **Wrong localStorage key** — `globalSetup` stored token as `'token'`; app reads `'arrgh_token'`. Fixed: all four keys (`arrgh_token`, `arrgh_username`, `arrgh_role`, `arrgh_allow_explicit`) now set correctly.
- **`waitForServer` polling `/api/version`** — returned 401 before #70 fix, never resolved. Fixed: uses `/api/auth/login` as ready-check (always responds).
- **`page.evaluate` SecurityError** — reading `localStorage` before page navigation. Fixed: uses `context().storageState()` instead.
- **Discover UI hits MangaUpdates** — library/download tests that added titles via the Discover search would be flaky in CI (live MangaUpdates calls). Fixed: all add-title flows use `POST /api/discover/add` API directly; only the Discover smoke test hits the UI.
